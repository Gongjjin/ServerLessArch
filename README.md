# ServerLessArch
## Gitmoji 커밋 규칙

Gitmoji를 사용하여 커밋의 성격을 이모지로 나타낼 수 있습니다. 주요 이모지와 설명은 다음과 같습니다:

- 🐛 `fix:`: 버그 수정
- 🎉 `feat:`: 새로운 기능 추가
- ✨ `feature:`: 새로운 기능이나 기능 개선
- 📚 `docs:`: 문서 변경
- 🚀 `perf:`: 성능 개선
- 🔧 `chore:`: 잡일 변경 (빌드 도구 변경 등)

## Git Flow 및 브랜치 전략

- `feature/`: 기능 개발 브랜치
- `bugfix/`: 버그 수정 브랜치
- `release/`: 배포 준비 브랜치
- `hotfix/`: 긴급 수정 브랜치
- `master/main`: 안정적인 코드가 있는 브랜치
- `develop`: 개발 중인 코드가 있는 브랜치
